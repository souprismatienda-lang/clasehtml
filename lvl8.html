Utilizando la propiedad width

Si establecemos la propiedad width a un porcentaje y la propiedad height en auto, la imagen será responsiva y aumentará o disminuirá su escala.

Utilizando la propiedad max-width

Si se establece esta propiedad al 100%, la imagen escalará hacia abajo si tiene que hacerlo, pero nunca escalará hacia arriba para ser más grande que su tamaño original.

Imagenes de fondo

Las imagenes de fondo también pueden responder al redimensionamiento y escalado. Vamos a ver 3 metodos diferentes:

Metodo 1

Si la propiedad background-size se establece en "contain", esta escalará e intentará encajar en el area de contenido. Sin embargo, la imagen mantendrá su relación de aspecto (la relación proporcional entre el ancho y alto de la imagen)

Metodo 2

Si la propiedad background-size se establece en "100% 100%" la imagen de fondo se estirará para cubrir el total del area que la contiene.

Metodo 3

Si la propiedad background-size se establece en "cover", la imagen de fondo escalara para cubrir el total del area de contendio, pero este valor mantiene la relacion de aspecto y algunas partes del fondo podrian cortarse.

Diferentes imagenes para diferentes dispositivos

Una imagen de alta resolucion puede ser perfecta para la pantalla de una PC, pero inutil en un dispositivo pequeño. ¿Por que cargar una imagen grande cuando tenenmos que escalara de todas maneras?. Para reducir la cara, o por cualquier otra razón, podemos utilizar los media queries para mostrar diferentes imagenes en diferntes dispositivos

El elemento <picture>

El elemento HTML <picture> nos da a los desarrolladores web más flexibilidad, al momento de especificar recursos de imagenes. El uso más común de este elemento es en imagenes utilizadas en diseño responsivo. En vez de tener una imagen escalada hacia arriba y abajo basada en el viewport, podemos diseñar multiples imagenes para llenar mas adecuadamente el navegador.

El atributo src es requerido, y define la fuente de la imagen.
El atributo media es opcional, y acepta las reglas del media query de CSS.
También deberiamos definir un elemento <img> para navegadores que no soportan el elemento picture
